# SCXML Studio

一个完全在本地浏览器中运行的 SCXML 1.0 编辑与可视化工具。SCXML 源码是唯一真实数据，状态图、结构树、属性检查器和问题列表都由源码实时派生。

## 功能

- Monaco XML 编辑器，编辑时实时显示 XML 与 SCXML 语义错误
- 普通状态、复合状态、并行状态、终态、历史状态及初始配置可视化
- 大型状态机按层级钻取显示；双击复合状态进入，使用面包屑返回
- 按转换流向自动分层布局：从初始状态向右排布，层内减少连线交叉；无连线时回退紧凑网格
- 当前层转换聚合显示，复合状态公共转换使用紧凑转换条呈现
- 子状态发出的跨层转换以虚线聚合到可见祖先上，悬停提示真实来源；双向转换自动错开不重叠
- 选中状态时，父/祖先状态定义的转换（SCXML 中对活动子状态同样生效）以紫色虚线从选中状态画出，悬停提示定义来源
- 深色/浅色主题一键切换，选择保存在本地下次自动生效；Monaco 编辑器与状态图配色同步切换
- 支持 `?edges=all&scope=<状态id>&theme=light` URL 参数直接打开指定层级、连线模式和主题
- 状态节点可拖拽定位，转换线会实时跟随
- 底部源码/问题面板可一键收起，让画布占满纵向空间；点击标签或定位问题时自动展开
- 转换的 `event`、`cond`、`target` 可视化和属性编辑
- W3C SCXML 命名空间、元素结构、ID/IDREF、初始状态、并行状态和属性约束检查
- 打开与保存标准 `.scxml` 文件
- XML 尚未输入完整时保留最后一次有效画布，不使用错误的中间模型覆盖视图
- 所有编辑器资源和 Worker 都打包在本地，不使用 CDN 或云服务

## 本地运行

需要 Node.js 22 或更高版本。

```bash
npm install
npm run dev
```

浏览器访问终端显示的本地地址，默认是 `http://127.0.0.1:5173`。

生产构建：

```bash
npm run build
npm run preview
```

## 测试

```bash
npm run check
```

`check` 同时执行 TypeScript 严格类型检查和全部单元/组件测试。测试重点覆盖非法 XML、命名空间、重复 ID、错误目标、初始状态范围、并行状态、多目标转换、null datamodel、属性互斥、历史状态表示以及编辑过程中的即时错误反馈。

## 标准边界

校验目标是 [W3C SCXML 1.0 Recommendation](https://www.w3.org/TR/scxml/) 的文档约束。当前版本实现编辑、结构校验与可视化；执行器、外部 I/O Processor、ECMAScript 表达式静态分析和 XSD 1.1 校验属于下一阶段。无法严格验证的自定义 datamodel 会显示警告，不会假装已经通过完整处理器合规测试。
